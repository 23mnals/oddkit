Staircase
Reveal

A full-screen solid cover made of N equal bands. On trigger, all bands slide to the same side — each delayed by a fixed offset, so their leading edges form a diagonal staircase sweeping across the screen.

<script>
  const reveal = new StaircaseReveal({
    bandCount:  11,
    coverColor: '#EBEBE5',
    staggerMs:  68,
    bandDur:    760,
    onRevealed() {
      // cover fully off — trigger hero intro here
      window.__heroIntro && window.__heroIntro();
    }
  });

  reveal.play();      // play on load (random direction)
  reveal.replay();    // replay on page transition
  reveal.play({ side: 'go-right', order: 'bottom' }); // forced direction
</script>

Page Transition — matchTarget

Pass matchTarget pointing at the destination section. On every replay(), the bands automatically read that element's background-color — no hardcoded colors needed.

Next page bg
#FFFFFF 
// matchTarget: auto-reads destination background on every replay
const reveal = new StaircaseReveal({
  matchTarget: '#next-section',  // CSS selector or DOM element
});
reveal.play();   // reads #next-section's backgroundColor → bands match
reveal.replay(); // re-reads every time — always in sync, no hardcoded colors

// Override for a one-off colour (ignores matchTarget that call only):
reveal.replay(null, '#0A0A0A');
How it works

A fixed cover fills the viewport with bandCount equal-height bands. On play(), every band gets the same slide-out class (go-left or go-right), but each has an increasing transition-delay — offset by staggerMs per band. This makes the bands' leading edges form a continuous diagonal, sweeping the cover off the screen in one uninterrupted motion. Direction randomises every call.

03

// Scroll-driven (axis: 'x'):

window.addEventListener('wheel', (e) => {
  reveal.replay({
    side:  e.deltaY > 0 ? 'go-down' : 'go-up',
    order: e.deltaY > 0 ? 'top'     : 'bottom'
  });
}, { once: true });

Scroll
Airflow Text

A fixed icon sits at the centre. As you scroll, any text row approaching the icon gets pushed sideways — label left, value right — by an invisible airflow. Rows spring back once the icon passes. The icon cycles automatically.

Preview
<script src="scroll-airflow-text.js"></script>

<!-- Fixed icon container -->
<div id="icon-stage" style="position:fixed;top:50%;left:50%;
     transform:translate(-50%,-50%);width:72px;height:72px;
     pointer-events:none;z-index:10">
  <div id="icon-slot" style="position:absolute;inset:0"></div>
</div>

<!-- Add data-push to any element you want pushed -->
<div class="row">
  <span class="label" data-push>Location</span>
  <span class="value" data-push>San Francisco</span>
</div>

<script>
new ScrollAirflowText({
  iconEl:    document.querySelector('#icon-slot'),
  icons:     ['icon-a.svg', 'icon-b.svg', 'icon-c.svg'],
  cells:     '[data-push]',
  sideOf:    el => el.classList.contains('label') ? -1 : 1,
  maxPush:    380,
  falloffY:   360,
  iconRadius: 110,
  ambient:    80,
  swapThresholdPx: 300,
});
</script>

How it works

On every scroll event, each registered cell's vertical distance to the icon is measured. A smoothstep curve maps that distance to a push strength (0 when far, 1 when overlapping). The push is always directed away from the icon — left for labels, right for values — and is capped at maxPush. An ambient baseline ensures even rows that wouldn't naturally overlap the icon still feel the wind. All displacement is applied as translate3d on the GPU compositing layer, so no layout is triggered.
