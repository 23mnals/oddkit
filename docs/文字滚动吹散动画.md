# 文字滚动吹散动画 · ScrollAirflowText

一个滚动驱动的「文字被气流推开」交互组件。中心一个固定 icon（可循环切换），随着用户滚动，**当某一行文字靠近 icon 时，会沿水平方向被推开**，让 icon 周围始终留出干净空白。文字离开 icon 影响半径后自动回到原位。

## 视觉效果

- 文字从两侧向中心移动 → 接近 icon 时被气流推开（label 推到更左、value 推到更右）
- 推力强度跟「行到 icon 的纵向距离」呈 smoothstep 衰减
- icon 永远不会被文字盖住（推力会按需要至少把文字推开到 icon 半径之外）
- 滚动累计到一定距离后，icon 切换到下一张（淡入淡出 + 微旋转）

## 文件

| 文件 | 用途 |
|---|---|
| `文字滚动吹散动画.js` | 组件源码，定义全局 `ScrollAirflowText` 类 |
| `文字滚动吹散动画.md` | 本文档 |

## 最小用法

```html
<!-- 1. 一个固定居中的容器作为 icon 槽位 -->
<div style="position:fixed; top:50%; left:50%;
            width:96px; height:96px;
            transform:translate(-50%,-50%);
            z-index:30; pointer-events:none;">
  <div id="iconSlot" style="position:absolute; inset:0;"></div>
</div>

<!-- 2. 任何带 [data-push] 的元素都会参与吹散 -->
<main style="padding: 70vh 0;">
  <div class="row" style="display:flex; gap:32px;">
    <span class="label" data-push>Address</span>
    <span class="value" data-push>北京市朝阳区 …</span>
  </div>
  <!-- 更多行 -->
</main>

<script src="文字滚动吹散动画.js"></script>
<script>
  new ScrollAirflowText({
    iconEl: document.getElementById('iconSlot'),
    icons: [
      'icons/heart-wings.svg',
      'icons/gift-hearts.svg',
      'icons/speech-heart.svg'
    ],
    cells: '[data-push]'
  });
</script>
```

## 配置项

| 参数 | 默认值 | 说明 |
|---|---|---|
| `iconEl` | `null` | **必填**。一个尺寸固定的容器元素。组件会向其内部插入两层 `<img>` 做交叉淡入。 |
| `icons` | `[]` | 图标资源数组（URL / 路径）。建议 ≥ 2 张才有循环切换效果。 |
| `cells` | `'[data-push]'` | 参与吹散的元素。可以是 CSS 选择器、`Element`、`NodeList` 或 `Element[]`。 |
| `sideOf` | `null` | `(el) => -1 \| +1`。自定义推开方向：返回 `-1` 推向左、`+1` 推向右。默认按元素中心相对视口中心自动判定。 |
| `iconRadius` | `110` | icon 周围预留的空白半径（px）。文字会被推开至少这个距离的两倍直径以外。 |
| `maxPush` | `380` | 单个元素最大横向位移（px）。 |
| `falloffY` | `360` | 纵向衰减距离（px）。行到 icon 的纵向距离超过这个值时，推力为 0。 |
| `ambient` | `80` | 在 icon 影响区里时的最小推力基线。让那些天然就已经避开 icon 的行也有可见的"被风吹一下"反应。 |
| `swapThresholdPx` | `300` | 滚动累计多少 px 后切到下一个 icon。 |
| `swapCooldownMs` | `700` | 两次 icon 切换的最小时间间隔，防止快速滚动时频繁闪。 |
| `swapInDuration` | `380` | icon 淡入持续时间（ms）。 |
| `swapOutDuration` | `520` | icon 形变/淡出持续时间（ms）。 |
| `swapEasing` | `cubic-bezier(0.22,1,0.36,1)` | icon 切换的缓动函数。 |

## 实例方法

```js
const flow = new ScrollAirflowText({ ... });

flow.refresh();   // 重新测量所有 cell 的静止位置（字体加载、布局变化后调用）
flow.update();    // 立即重算一次推力（一般无需手动调）
flow.swap();      // 立即切换到下一个 icon
flow.destroy();   // 解绑监听并把 cell 的 transform 清干净
```

## DOM 约定与注意事项

1. **iconEl 必须有自己的尺寸和定位**。组件只插入两层 absolute 充满父容器的 `<img>`，不会替你管 `position: fixed` / 居中。最常见的用法是把 iconEl 放进一个 `position: fixed; top:50%; left:50%; transform: translate(-50%,-50%)` 的舞台里。
2. **cell 元素请不要再叠加自己的 CSS `transform` / `transition`**。组件用 inline transform 实时改写位置，外部 CSS 的 `transform` 会被覆盖；如果挂了 `transition: transform`，预览类 iframe 环境里可能会把动画卡住。
3. **页面需要有上下滚动空间**才能让行经过 icon。如果只有半屏内容，给 `<main>` 加 `padding: 70vh 0;` 之类。
4. **响应式**：组件会监听 `resize` 自动 `refresh()`；如果用 JS 动态增/删行，再调一次 `flow.refresh()` 即可。
5. **方向控制**：默认按 cell 几何中心 vs. viewport 中心来判定向左推还是向右推。如果是上下排版（label 在上、value 在下）或者特殊场景，传 `sideOf: el => el.classList.contains('label') ? -1 : 1` 显式指定。

## 调参建议

| 想要的感觉 | 改哪些 |
|---|---|
| **更夸张、文字飞得更远** | ↑ `maxPush`、↑ `iconRadius`、↑ `falloffY` |
| **更含蓄、只有最近的行有反应** | ↓ `falloffY`、↓ `ambient` |
| **icon 切换更密集** | ↓ `swapThresholdPx`、↓ `swapCooldownMs` |
| **icon 切换更稳重** | ↑ `swapThresholdPx`、↑ `swapInDuration` / `swapOutDuration` |

## 性能

- 所有计算都跑在 `scroll` 事件里，开销 = 每个 cell 一次距离平方 + 一次 smoothstep。
- inline transform 走 GPU 合成层，不会触发 layout。
- 不依赖 `requestAnimationFrame` 或 `setInterval` 物理循环；用户的滚动手势本身提供动画时序。
